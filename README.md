## 工夫した点

- 要求仕様書や設計資料とはまず何か・どのように記載するかを一から調べたり、
講師に質問し進めていったこと

- 要求仕様書: 抽象レベルを意識し、最も大切な使用の部分をただ全部機能を導入するのではなく、
導入する理由や導入しない理由の両方から考え作成したこと

- 画面仕様書: 基本はシンプルに分かりやすいUIを意識し、画面と矢印で画面の動きが第三者でも
理解できるように意識し作成したこと、また要求仕様書と同じことはできるだけ記載しないなど

- 設計資料:
状態遷移図は「状態」を、クラス図は「構造」を、シーケンス図は「流れ」を設計の面から
実装を補助する資料という事を意識。
・状態遷移図は「状態」と「画面」を切り分けて考えること。
・クラス図では依存関係を意識することで、実装を想定し保守性・拡張性にできるだけ優れるように作成したこと。
・シーケンス図は関数の粒度を合わせることや処理の流れにおいてどんな戻り値が返ってきてどんな処理を行うかを
意識し作成したこと。

- 実装: オブジェクト指向でのカプセル化を意識し、リファクタリング時の影響範囲を分かりやすくした。
また拡張のしやすさやエラーの防止にも努めて実装していきました。
エラーが出た場合、コンソールからファイルやエラーの種類を仮定・予想・実践を繰り返し、少しずつ
デバック対応できるように努めたこと。
